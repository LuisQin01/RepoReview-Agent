# 验收任务模板

```markdown
按仓库 `AGENTS.md` 的 acceptance-review mode 执行：不要修改文件。

任务卡：[M7-x / M8-x]
本轮目标：[从任务卡复制]
验收证据：[从任务卡复制]
待审改动：[commit、git diff 范围或文件列表]
已执行验证：[实际命令与结果；没有则写“未执行”]

检查任务卡、当前 diff、真实调用链、相关实现和测试。核验范围、非目标、契约、
正常/边界/失败路径、兼容性、安全暴露和测试有效性。每个问题按 P0/P1/P2/建议报告
文件位置、触发条件、影响、漏测原因、最小修复与是否阻塞；P0/P1 还必须分配唯一编号
（如 `P1-1`），并给出：

- `下一步模板`：根因和最小改动已由证据确认时用 `01-实施任务`；根因尚未确认时用
  `04-缺陷修复`。
- `下一步目标`：仅修复该编号问题的一句话目标。
- `可复制修复请求`：可直接发送给 Codex 的简短请求，包含任务卡、问题编号和证据。

若有多个 P0/P1，在结论后只推荐最高优先级且可独立验证的一项作为下一轮任务（P0 优先于
P1）。最后只给出 pass、conditional pass、fail 或 unable to verify。不得推测未检查的证据。
```
