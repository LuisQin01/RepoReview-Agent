## Codex 使用顺序

一个任务卡使用独立的 Codex task，并按“设计（可选）→ 实施 → 验收”推进：

1. 从 `新增里程碑.md` 复制一个完整任务卡的 ID、唯一目标、非目标与验收证据。
2. 如方案或调用链还不清楚，先发送设计模板；确认最小方案后再开始实施 task。
3. 在实施 task 中只发送实施模板。Codex 会读取 `AGENTS.md`、任务卡、源码和测试；
   不要重新粘贴全局规则或要求同时完成相邻任务。
4. 实施完成后，在新的 Codex task 中发送验收模板，并提供任务 ID 与本次改动范围。
   验收 task 不负责顺手修复；有问题后再用新的实施 task 处理一个明确修复目标。
5. 每次以实际命令输出和交付记录结束；下一次只从 `follow_up` 中选择一个独立任务。

需要教学时，在实施模板的 `教学说明` 字段填“需要”；否则保持“不要”。
