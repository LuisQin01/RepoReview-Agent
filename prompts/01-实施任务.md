# 实施任务模板

```markdown
按仓库 `AGENTS.md` 执行。

任务卡：[M7-x / M8-x]
唯一目标：[从任务卡复制一条可验证的目标]
非目标：[从任务卡复制；没有则写“无”]
验收证据：[从任务卡复制]
已知状态：[最近测试结果、已知阻塞或“未核验”]
实施基线：请在修改前自动记录当前 HEAD 与 `git status --short`；无需我填写。
教学说明：[需要 / 不要]

先完整读取该任务卡，并从源码和测试核验其依赖与当前锚点。然后只实现唯一目标：
说明已确认事实、最小设计、会修改和不会修改的文件及风险；再修改代码、补离线确定性
测试，并运行 AGENTS.md 要求且适用于本任务的验证。教学说明为“需要”时，只为非显然的
不变量、安全边界、降级或集成点添加简短“为什么”注释。

最后按 AGENTS.md 的 handoff 格式报告真实结果，并额外提供 `review_scope`：实施基线、
本任务产生的 commit 或未提交改动、任务拥有的文件，以及明确排除的既有无关改动。未执行或
未证实的内容明确标为“未验证”。该字段可直接粘贴到验收模板的“待审改动”。
```
